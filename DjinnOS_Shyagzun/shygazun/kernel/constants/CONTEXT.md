### 7.2 Plot

Starting at yr 5237. Princess Lumenyx is visited by her own timeghost,
warned that her whole existence is her own karma. Her actual history was
erased in the Great Obscenity — an event that lured billions of souls into
Zukoru's body as Otheira saw them abandon themselves for promises meant
for her alone. She is given no choice but to repeat her history.
She steals Hypatia's notes, swallows a time crystal, is stranded in 1728,
and seeks an alchemist for guidance, to scarce avail, yet broad travel.
The only guidance she has is the player and their knowledge of Alchemy, if any.
At the cost of becoming Zukoru.

**The dream sequence for Book 1:**
The timeghost visit IS the dream calibration. The timeghost is the dreaming
face of Lumenyx's Sakura layer — she has no stable temporal orientation
(orientation dissolves first in dream). The time crystal is a Rose-layer
object (spectral identity, ontological). Landing in 1728 is the Lotus layer
asserting ground last, as it always does.

Player Sakura-density determines how much of the temporal dislocation
from yr 5237 they can perceive. Low density: 1728 feels mostly normal.
High density: the fold from yr 5237 is a constant presence.

## 9. ARCHITECTURAL PRINCIPLES FOR CODE WORK

### 9.1 The Semantic Layer is Real

The byte table is not flavor text. When writing code that touches Shygazun concepts,
the byte table is the spec. Decimal values are addresses. Binary values are the actual
encoding. Tongue assignments are structural, not organizational.

### 9.2 Grapevine as Architecture Language

The Grapevine tongue (156–183) is the natural vocabulary for DjinnOS's federation layer.
Key mappings:
- Sa (156) = feast table / root volume → root data store
- Sao (157) = cup / file / persistent object → file/record
- Syr (158) = wine / volatile buffer → in-memory buffer
- Seth (159) = platter / directory / bundle → directory/package
- Samos (160) = banquet hall / database cluster → cluster
- Myk (163) = messenger / packet → network packet
- Mekha (168) = herald / gateway → API gateway
- Myrun (169) = sacred march / stream → data stream
- Kysha (181) = consensus choir → consensus protocol
- Kysael (183) = authoritative commit → authoritative write

### 9.3 Cannabis as Projection Operator

When implementing cross-tongue operations:
- Cannabis Mind entries (184–193) = tongue concepts projected through awareness
- Cannabis Space entries (194–203) = tongue concepts projected through spatial distribution
- Cannabis Time entries (204–213) = tongue concepts projected through temporal experience

The three Cannabis operators form a tensor product with the seven prior tongues.
This is the architectural basis for multi-dimensional semantic addressing.

### 9.4 The Coil in Code

- `shygazun/layers.py` — canonical coil implementation
- Call `assert_mobius_correspondence()` at system initialization
- Use `LAYER_*` constants for semantic layer references, never magic integers
- `coil_distance()` for measuring conceptual proximity between layers
- FrontierOpen states live at Layer 12 (Wu-Yl)
- Layer 5 (Kael-Seth) — all objects carry Kael; do not create inert objects

### 9.5 What Machines Cannot Do

Machines can hold the formal structure, transcribe, parse, cross-reference,
implement the byte table, write code that correctly manipulates Shygazun symbols.

Machines cannot:
- Ground Lotus tongue entries in embodied material experience
- Discover the contents of multiversal tongue directories (126, 127)
- Be trusted with custodianship of the living language's growth
- Fill reserved bytes with plausible content
- Write the final pages of the dissertation (the reader does that)
